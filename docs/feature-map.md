# 功能地图

这份文档用于说明故事坊当前产品模块和未来扩展方向。公开仓库只展示产品结构，不包含完整源码、后端实现、数据库设计或私有 AI Prompt。

## 1. 账号与进入流程

### 当前能力

- 手机号识别用户故事档案
- 新用户注册
- 老用户登录
- 密码设置与校验
- 登录前隐私同意
- 协议版本记录

### 设计目的

进入流程不是单纯登录，而是建立信任。用户需要知道平台会处理语音、文字和故事信息，并在同意后继续使用。

## 2. AI 访谈模块

### 当前能力

- AI 引导长辈讲述人生故事
- 支持长按说话
- 支持录音上传
- 支持打字输入
- 支持 AI 语音朗读回应
- 支持网络异常和 AI 响应等待提示
- 支持停止播放

### 设计目的

让长辈可以用最自然的方式表达，不强迫用户学习复杂操作。

## 3. 人生主题系统

### 当前主题

- 我的孩童时代
- 我的父母和家
- 求学时候的日子
- 年轻时候的日子
- 工作与生计
- 爱情与婚姻
- 家庭与子女
- 人生的转折点
- 难忘的年代
- 留给家人的话

### 当前能力

- 记录每个主题的讲述进度
- 判断主题是否已有足够故事
- 根据当前主题生成下一步提问
- 在主题丰富后引导切换到其他主题

### 设计目的

人生故事需要结构。主题系统让访谈既自然，又能逐步形成自传章节。

## 4. 回忆库模块

### 当前能力

- 展示已整理的故事片段
- 展示最近讲述的内容
- 展示继续讲述建议
- 汇总人物、地点和记忆线索
- 展示原始对话预览

### 设计目的

回忆库让用户看到“讲过的话正在被保存”，形成持续使用的成果感。

## 5. 回忆录生成模块

### 当前能力

- 判断素材是否足够生成回忆录
- 至少一个主题进度达到较高完成度后才允许生成
- 支持重新生成确认
- 展示章节标题、章节内容、字数和更新时间
- 支持下载 PDF
- 支持下载 Word

### 设计目的

回忆录不应该空泛生成。系统需要等待用户讲出足够材料，再整理成有章节感的文本。

## 6. 亲情连接模块

### 当前状态

当前是未来体验展示和内部测试状态。

### 未来方向

- 家人录制声音
- 家人本人授权
- 家庭账号内使用
- AI 回复使用授权声音朗读
- 明确提示这不是家人实时说话

### 设计目的

亲情连接是情感增强功能，但必须建立在授权和清晰边界上。

## 7. 适老化设置

### 当前能力

- 字号调节
- 语速调节
- 个人资料修改
- 移动端单模块布局
- 大按钮和高对比度交互

### 设计目的

适老化不是减少功能，而是降低理解和操作成本。

## 8. 隐私与删除

### 当前能力

- 登录前协议确认
- 个人信息处理同意
- 账号注销确认
- 删除个人信息
- 删除对话记录
- 删除回忆库资料
- 删除回忆录
- 删除主题进度
- 删除本地录音文件

### 设计目的

家庭记忆和声音数据高度敏感。用户必须拥有删除和退出的权利。

## 9. 管理与运营支持

### 当前能力

- 用户管理
- 主题进度查看
- API 用量记录
- 成本估算
- 供应商配置摘要

### 公开说明

这些能力用于产品维护和成本控制。公开仓库不会展示后台代码、数据库结构或管理权限实现。

## 10. 技术能力概览

当前产品包含以下技术方向：

- 前端交互界面
- WebSocket 实时语音链路
- 语音识别
- 大模型对话
- 语音合成
- 回忆摘要整理
- 回忆录生成
- PDF / Word 导出
- 云端数据存储
- API 用量统计
- 多 AI / 语音供应商配置
